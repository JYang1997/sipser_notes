<h1 id="random-notes-about-language">Random notes about language</h1>
<h3 id="section">1.</h3>
<pre><code>There are three different terms,
1. language
2. language generator (grammar of the language)
3. language recognizer (machine (automaton))
</code></pre>
<ol>
<li><strong>language</strong> are represent in the form of expressions (regular expression, context-free expression)</li>
<li><strong>grammar</strong> specified rules for generating the language.</li>
<li><strong>automaton</strong> is a machine that recognize the language.</li>
</ol>
<h3 id="section-1">2.</h3>
<p>Pumping lemma is useful for proving a language is not CFL or RL.</p>
<p>Basic Idea:</p>
<ol>
<li>If the language size is <strong>finite</strong>, then it is obvious. (RL)</li>
<li>If the language size is not <strong>finite</strong>, then by pigeonhole, there must be part of the rule reused.</li>
<li>This reuse of rule lead to loop in the path, then remove such loop, or repeat multiple times should not affect acceptance of the string.</li>
</ol>

<p>The proof is really trivial (based on pigeonhole principle).</p>
<p>The pumping lemma is useful when you try to prove a language is not regular.<br>
But this theorem is not really useful if you try to prove a language <strong>is</strong> regular.<br>
The pumping lemma is a one way statement, says nothing about non-regular language.<br>
Showing second part is not sufficient for proving the language is regular.</p>
<p>To prove a language is regular, one can start from the definition, or its equivalent models.</p>

<p>Turing Machines computation process are represented by <strong>configuration</strong>. Configuration describes TM’s current state, the current tape contents, and the current head location. The transition function <span class="katex--inline"><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>δ</mi></mrow><annotation encoding="application/x-tex">δ</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 0.69444em; vertical-align: 0em;"></span><span class="mord mathdefault" style="margin-right: 0.03785em;">δ</span></span></span></span></span> describes how TM’s changes from one configuration to another. We say configuration 1 <strong>yields</strong> configuration 2, if there is transition function that go from 1 to 2.</p>

<h4 id="decidable-v.s.-recognizable">Decidable v.s. Recognizable</h4>
<blockquote>
<p><strong>Definition</strong><br>
Call a language <strong>Turing-recogniziable</strong> if some Turing machine recognizes it.</p>
</blockquote>
<blockquote>
<p><strong>Definition</strong><br>
Call a language <strong>Turing-decidable</strong> or simply <strong>decidable</strong> if some Turing machine decides it.</p>
</blockquote>
<p>The difference between deciadable and recognizable is that, for Turing decidable, there are a turing machine that recognize such language and it always accept or reject, never <strong>halts</strong>.<br>
Turing-Recognizable only guarantee that if string is in such language than turing machine accepts it, said nothing about what happen if machine does not accept it. (Both halt and reject is possible).</p>
<h4 id="robustness-of-turing-machines">Robustness of Turing Machines</h4>
<p>The Turing machine have very high level of robustness. This basically means that the original turing machine model have equivalence expressing power (recognize) compare to its reasonable variants (those variants may seems more powerful, but we can show that all the add-on “features” can be simulated with original turing machine.)</p>
<p>Here are some example variants that are equivalence to the Turing machine.</p>
<ol>
<li>multitape turing machine (Thm 3.13)</li>
<li>non-deterministic Turing machine (Thm 3.16)</li>
<li>stay put turing machine.</li>
</ol>
<p>In general, one can demonstrate the equivalence by showing that the variant part can be simulates using plain Turing Machine. (see section 3.2, for details)</p>

<p>Theorem 4.5 state that determine whether two DFAs are equivalent is decidable.<br>
The proof idea:<br>
If DFA <em>A</em> and <em>B</em> is not equivalent, then there must be strings that only accept by either A or B.  (known as <strong>symmetric difference</strong> of <em>A</em> and <em>B</em>).</p>
